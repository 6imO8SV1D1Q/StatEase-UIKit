
# 📘 統計検定2級 初学者向け学習アプリ  
**要件定義書 v1.0（完全版）**

---

# 0. この文書について
本書は「統計の初学者がつまずかずに統計検定2級レベルを習得できるアプリ」の  
**MVP（Minimum Viable Product）要件**を定義したもの。

- 対象：個人開発（iOS/ UIKit）
- 想定ユーザー：数学に苦手意識があり、短時間で理解したい初学者
- 目的：書籍・動画・AI学習の欠点を埋める新しい学習体験を作る

---

# 1. プロジェクト概要

## 1.1 目的
「統計が苦手な人でも“理解できた感覚”を得られる学習アプリ」を提供する。

## 1.2 このアプリで解決したい課題
既存教材の問題：
- 文章が長い・難しい  
- 動画は長くて管理しにくい  
- 初学者がつまずくポイントが分からない  
- 続かない  
- AIに質問しても難しい回答が返る  

## 1.3 アプリのコンセプト
**「5分で1つ理解できる、動く統計」**

---

# 2. ターゲット

- 数学に苦手意識がある社会人・大学生  
- 忙しくてまとまった時間が取れない人  
- 読書や動画学習だと挫折した人  
- 基礎から丁寧に学びたい人  

---

# 3. 差別化ポイント（本アプリの強み）

## 3.1 説明と図が同期する「アルゴリズム図鑑方式」
スクロールに合わせて
- 説明  
- 静止図  
- 3〜6秒の短いアニメ  
が **同じステップに配置され、流れに沿って動く**。

→ 書籍・動画・AIのいずれでも再現できないUX。

## 3.2 Manim（3Blue1Brown系）で生成した高品質アニメ
- 数式の意味を動きとして示す  
- ステップごとに短く分割し、視覚的負荷を減らす  
- 「アルゴリズム図鑑」スタイルのミニアニメ

## 3.3 「やさしい結城浩風」の統一文体
- 専門用語の乱発禁止  
- 無理な比喩禁止  
- 1ステップ＝短文＋図  
- 初学者でも誤解しない書き方に統一

## 3.4 コンテキスト限定AI質問機能
- レッスンの範囲外の話はしない  
- 回答は**300字以内**  
- 文体も「やさしい結城浩風」で統一  
- チャットAI特有の“暴走説明”を抑制

## 3.5 忙しい人向けのUX設計
- 1レッスン5分  
- 目安時間表示（1分〜5分）  
- 完了チェックのみ（ストレスレス）  

---

# 4. 学習設計（LX）

## 4.1 レッスンの最小単位（5基準で決定）
1. **認知負荷が低いか**  
2. **依存関係が切れるか**  
3. **アニメで1つの動きを示せるか**  
4. **初学者が1つの概念として理解できるか**  
5. **5分以内に収まるか**

### 最小単位の例（記述統計）
- 平均  
- 分散  
- 標準偏差  
- 代表値の比較  
- ばらつきの直感  
など

## 4.2 レッスン構造（全文テンプレ）

### **① 結論**
「これは〇〇を表す量です。」

### **② 意味（何を表しているか）**
比喩は禁止、短く正確に。

### **③ 手順（計算方法）**
箇条書き＋1行式。

### **④ 例（具体値）**
理解の確認。

### **⑤ 1行まとめ**
認知負荷を下げる。

---

# 5. コンテンツ構造

## 5.1 章構成（統計検定2級準拠 + 初学者順序）
1. 記述統計（最重要・最初に理解する）  
2. 確率  
3. 分布（正規分布 → χ²分布 → t分布 → F分布）  
4. 標本分布  
5. 推定  
6. 検定  
7. 回帰分析  

## 5.2 レッスン一覧（例：初期MVP）
**第1章：記述統計（9レッスン）**
- 平均
- 分散
- 標準偏差
- 代表値の特徴
- ヒストグラム
- 算術平均と加重平均
- 変動係数
- データのばらつきの直感
- 1章まとめ

---

# 6. アニメーション仕様（Manim）

## 6.1 基本方針
- 1ステップ＝1クリップ（3〜6秒）
- 音声なし
- 色は2色以内（混乱防止）
- 静止図と動作は同じレイヤー構造を共有

## 6.2 テンプレ構造
以下を一貫して使う：

1. 無地背景  
2. 文字（最小限）  
3. 図形（点・線・棒・分布）  
4. 簡易数式（本質のみ）  
5. アニメーション（移動・拡大・合成など）

---

# 7. レッスン画面UI仕様（UIKit）

## 7.1 スクロール構成
- UITableView（または UIScrollView）で**ステップごとに縦並び**
- 各セルは
  - テキスト
  - 画像
  - 動画（AVPlayerLayer）
  のいずれか

## 7.2 レイアウト
- 左揃えのテキスト  
- 図は中央揃え  
- 動画はタップ再生（自動再生なし）

---

# 8. 確認問題
- 4択問題  
- レッスンとは別記事  
- 簡易解説あり  
- 過去問は使用せず、オリジナルで作成  
- 回答時間は短く（1問約20秒目安）

---

# 9. 進捗管理（MVP）

- 完了チェック（表示のみ）  
- 進捗バー（任意、低優先度）  
- ストリークは採用しない

---

# 10. AI質問機能

## 10.1 仕様
- 各レッスン末尾に「質問してみる」ボタン
- 初期は非表示、タップでチャットUIへ遷移
- レッスンのテキストをコンテキストとして渡す

## 10.2 制約
- 回答は**300文字以内**  
- 文体は**やさしい結城浩風**  
- レッスンの範囲外の話は禁止  

## 10.3 入力UI
- 画面下部に固定  
- UITextView + 送信ボタン  
- 過去質問の保存はしない（MVP）

---

# 11. 技術要件（iOS / UIKit）

## 11.1 対応
- iPhone（iOS 16+）
- UIKitのみ（SwiftUIは使わない）

## 11.2 画面
- UIKit LifeCycle（AppDelegate + SceneDelegate）
- UICollectionView / UITableView を中心に構成

## 11.3 データ管理
- レッスン本文：JSON（アプリ内同梱）
- 動画：mp4（同梱）
- 設定値：UserDefaults
- APIキー：Keychain

## 11.4 AI通信
- HTTPS（OpenAI API）
- 非同期でレスポンス表示

---

# 12. 非機能要件
- レッスン本文とアニメは完全オフライン  
- AI質問のみオンライン  
- レスポンス速度：2秒以内目標  
- 誤操作してもデータが消えない  

---

# 13. MVPスコープ（最小構成）

1. 記述統計（9レッスン）  
2. レッスンステップ構造  
3. Manimアニメ（各レッスン2本程度）  
4. 4択問題（章末5問）  
5. AI質問機能  
6. 設定画面（APIキー登録）  
7. 完了チェック
